天翼云 Logpush Backfill 指南

这是最终版轻量 backfill 方案。它会把指定时间窗内的 R2 历史日志重新取出，过滤 Worker 子请求，转换为天翼云要求的格式，并按明确限速发送到客户接收端。

架构

R2 logs/
-> parse-queue-backfill
-> Parser
-> processed-backfill/<run-id>/
-> send-queue-backfill
-> Sender
-> customer endpoint
热路径里不再使用 Durable Object。 每个 raw 文件独立解析，并直接写成最终 batch 文件。

这套方案保证什么

  • 精确回放 [BACKFILL_START_TIME, BACKFILL_END_TIME] 内的日志
  • 只发送父请求：ParentRayID = "00" 且 WorkerSubrequest != true
  • 使用和生产一致的 CTYun 鉴权 secrets 做 POST
  • 通过 queue 并发 + 200ms invocation floor 对 sender 做显式限速
  • run 完成后自动清理临时 batch 文件

投递语义

重要：这套方案是 at-least-once，不是 exactly-once。 如果客户 POST 已成功，但 Worker 随后没能写下 .done 标记，那么这个 batch 会被重试。 如果客户侧要求严格去重，需要按 batch identity 在接收端做 dedupe。

部署

字段含义
BACKFILL_START_TIME补传开始时间，ISO 8601
BACKFILL_END_TIME补传结束时间，ISO 8601，且必须是过去时间
BACKFILL_ENABLED必须显式设成 true 才会运行；checked-in 默认值是 false
BACKFILL_RATE每分钟最多 enqueue 多少个 raw 文件
BATCH_SIZE每次 POST 最多多少条日志，默认 1000
SEND_TIMEOUT_MSsender 等客户 ACK 的最长时间
wrangler queues create parse-queue-backfill
wrangler queues create send-queue-backfill
wrangler queues create parse-dlq-backfill
wrangler queues create send-dlq-backfill

wrangler secret put CTYUN_ENDPOINT --config wrangler-backfill.toml
wrangler secret put CTYUN_PRIVATE_KEY --config wrangler-backfill.toml
wrangler secret put CTYUN_URI_EDGE --config wrangler-backfill.toml

wrangler deploy --config wrangler-backfill.toml

调速

当前 checked-in 的 sender 理论 ceiling：

max_concurrency = 20
max_batch_size = 1
MIN_SENDER_INVOCATION_MS = 200
BATCH_SIZE = 1000

理论上限
= 20 * 1 * (1000 / 200) * 1000
= 100,000 行/秒

这个上限默认假设 batch 接近 1000 行，且客户 ACK 小于 200ms。

目标 ceiling推荐 sender max_concurrency
10,000 行/秒2
20,000 行/秒4
50,000 行/秒10
100,000 行/秒20

观察

curl https://ctyun-logpush-backfill.<your-subdomain>.workers.dev/backfill/status
wrangler tail ctyun-logpush-backfill

运维上请以 queue backlog 和 sender 日志为准。sender 日志里会带 ack_ms 和 queue_wait_ms。

什么时候算结束

最稳妥的判断条件：

  • /backfill/status 显示 status = done 或 cleaned
  • send-queue-backfill backlog 归零
